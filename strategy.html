Angular Technical Questions

Explain Change Detection in Angular — Default vs OnPush.
Default CD checks every component on any event. OnPush only checks when @Input references change, Observables emit (async pipe), or markForCheck() is called. For performance: use OnPush + Signals in Angular 16+. Always use trackBy in *ngFor. With Signals (Angular 17+), fine-grained reactivity eliminates most manual CD concerns.
What are Angular Signals and how do they differ from RxJS?
Signals are synchronous reactive primitives built into Angular core (v16+). They hold a value and notify dependents when it changes — like a computed() derived signal. RxJS is stream-based (async, time-based, multi-event). Signals are better for local UI state; RxJS excels at complex async data flows, HTTP, event streams. In Angular 17+, toSignal() bridges Observables to Signals.
How would you architect an NgRx store for a large enterprise application?
Use feature-level stores with createFeature(). Separate concerns: Actions (what happened), Reducers (pure state transitions), Effects (side effects/HTTP), Selectors (memoised derived state). Use Entity Adapter for list management. Implement strict typing. For very large apps, consider facade pattern to hide store complexity from components.
Explain SSR with Angular Universal / Angular 17 SSR.
Angular 17+ has native SSR built in (no separate Universal package). Server renders the initial HTML for fast FCP/SEO. Hydration (non-destructive in v17) reattaches event listeners without re-rendering DOM. Key considerations: avoid browser-only APIs (window/document) in shared code, use isPlatformBrowser(), handle TransferState for server-to-client data transfer, configure cache headers correctly.

RxJS Questions

When would you use switchMap vs concatMap vs mergeMap vs exhaustMap?
switchMap: cancels previous inner observable (search autocomplete). concatMap: queues, processes in order (sequential HTTP requests). mergeMap: all run in parallel (upload multiple files). exhaustMap: ignores new source until current completes (login button — prevents duplicate submissions). Most interview-critical: switchMap for search, exhaustMap for form submit.

System Design Questions

Design a Micro Frontend architecture for an enterprise Angular application.
Use Module Federation (Webpack 5) or Native Federation (Angular). Shell app loads remote modules at runtime. Each team owns an independent Angular app deployed separately. Share dependencies (Angular core) via shared scope. Communicate via CustomEvents or a shared EventBus service. Design: Shell → loads Header MFE, Dashboard MFE, Settings MFE. Lazy-loaded routes. Independent deployments. Separate NgRx stores per MFE or shared state via a root token.
How do you implement a robust authentication system in Angular?
JWT-based auth: store tokens in memory (not localStorage — XSS risk). Use HttpInterceptor to attach Bearer token to every request. Implement token refresh via silent refresh with a dedicated refresh Observable. Route Guards (canActivate) for protected routes. Auth facade service wrapping NgRx auth store. Handle 401 responses in interceptor by triggering refresh, then retrying original request using RxJS retryWhen or catchError with switchMap.

Performance Questions

How would you optimise a slow Angular application?
Audit with Chrome DevTools / Angular DevTools. Key strategies: (1) Lazy loading all routes, (2) OnPush change detection everywhere, (3) Pure pipes instead of component methods, (4) trackBy in ngFor, (5) Virtual scrolling (CDK) for large lists, (6) SSR for initial load, (7) Image optimisation with NgOptimizedImage, (8) Bundle analysis with webpack-bundle-analyzer, (9) Preloading strategies (PreloadAllModules or custom), (10) HTTP caching + request deduplication.
